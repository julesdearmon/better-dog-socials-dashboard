# Project Working Rules

## Keep The Brand Connection SOP Current

When working on platform connection, API credentials, developer app setup,
GitHub secrets, OAuth, app review, URL verification, or dashboard deployment
steps, check whether the process in `docs/BRAND-CONNECTION-SOP.md` changed.

If the user learns a new step, hits a new error, or a platform screen/process
changes, update `docs/BRAND-CONNECTION-SOP.md` in the same work session.

Never add access tokens, refresh tokens, client secrets, passwords, or clipboard
contents to the SOP or any committed file.
